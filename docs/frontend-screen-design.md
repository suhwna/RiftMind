## 1. 목적

이 문서는 RiftMind 웹 프론트의 초기 화면 구조와 사용자 흐름을 정의한다.

현재 백엔드는 아래 기능을 이미 제공한다.

- 소환사 프로필 동기화/조회
- 최근 경기 목록 조회
- 매치 상세 조회
- Elasticsearch 기반 조건 검색

초기 프론트는 이 기능을 전부 노출하는 것보다
`소환사 검색 -> 최근 경기 -> 경기 상세` 흐름을 먼저 완성하는 데 집중한다.

---

## 2. 권장 프론트 스택

- `Vite`
- `React`
- `TypeScript`
- `React Router`
- `TanStack Query`
- `Tailwind CSS`

### 선택 이유

- 현재 백엔드는 API 중심 구조라 `SPA`가 잘 맞는다.
- SEO/SSR이 핵심 요구사항이 아니므로 `Next.js`보다 가볍게 시작할 수 있다.
- `TanStack Query`로 API 호출, 로딩, 에러, 캐시를 깔끔하게 관리할 수 있다.

---

## 3. 제품 UX 원칙

- 메인 UX는 `검색`이 아니라 `회고를 위한 경기 탐색`이다.
- 사용자는 먼저 소환사를 찾고, 최근 경기 목록을 본 뒤, 특정 경기를 상세하게 본다.
- 고급 검색은 보조 기능으로 두고 메인 화면을 복잡하게 만들지 않는다.
- 초기 화면은 수치 나열보다 `빠르게 훑고 눌러볼 수 있는 카드형 구조`를 우선한다.

---

## 4. 라우트 구조

```text
/
  -> 홈 / 소환사 검색

/summoners/:puuid
  -> 소환사 대시보드

/matches/:matchId?puuid=:puuid
  -> 매치 상세

/search
  -> 고급 검색 (선택 노출)
```

### 라우트 역할

- `/`
  - Riot ID 입력
  - 동기화 시작
  - 최근 조회한 소환사 재진입
- `/summoners/:puuid`
  - 소환사 프로필
  - 최근 경기 목록
  - 간단 필터
- `/matches/:matchId`
  - 해당 경기의 참가자 상세
  - 아이템, 룬, 스펠, 딜량, CS, 시야 확인
- `/search`
  - 고급 검색 실험 공간
  - 초기 메인 네비게이션에서는 비중을 낮춘다

## 5. 화면 구성

## 5.1 홈 / 소환사 검색 화면

### 목표

- 사용자가 Riot ID로 소환사를 찾아 첫 진입한다.

### 주요 요소

- 서비스 헤더
- Riot ID 입력 폼
  - `gameName`
  - `tagLine`
- `전적 불러오기` 버튼
- 최근 조회한 소환사 목록
- 현재 서버 상태를 보여주는 작은 배지
  - gateway 연결 여부
  - 동기화 진행 상태

### API 연결

- `POST /api/v1/summoners/sync`
- `GET /api/v1/summoners/by-riot-id`

### UX 메모

- 입력은 `Hide on bush#KR1` 한 줄 입력도 가능하게 처리하는 것이 좋다.
- 동기화 성공 시 `/summoners/:puuid`로 이동한다.
- 동기화 중에는 버튼을 비활성화하고 진행 상태를 보여준다.

---

## 5.2 소환사 대시보드 화면

### 목표

- 한 소환사의 최근 경기 흐름을 빠르게 훑고, 특정 경기를 선택하게 만든다.

### 상단 영역

- 프로필 아이콘
- Riot ID
- 소환사 레벨
- 마지막 동기화 시각
- `다시 동기화` 버튼

### 요약 영역

- 최근 경기 수
- 최근 승/패 수
- 주 챔피언
- 자주 가는 포지션

이 영역은 초기에는 단순 집계로 시작하고,
4주차 패턴 분석이 들어오면 확장한다.

### 최근 경기 목록

각 카드에 표시할 정보:

- 챔피언 한글명
- 큐 이름
- 포지션 한글명
- K / D / A
- 승패
- 게임 시작 시각

### 간단 필터

초기 메인 노출 필터:

- 챔피언
- 포지션
- 승패
- 큐

### API 연결

- `GET /api/v1/summoners/{puuid}`
- `GET /api/v1/summoners/{puuid}/matches`

### UX 메모

- 기본 정렬은 최신순
- 목록에서 카드를 누르면 `/matches/:matchId?puuid=:puuid`로 이동
- 검색 필터는 상단 드롭다운 또는 토글 칩 형태가 적절하다

---

## 5.3 매치 상세 화면

### 목표

- 한 경기의 참가자 정보를 회고용으로 확인한다.

### 헤더 영역

- 큐 이름
- 게임 시작 시각
- 게임 길이
- 게임 버전

### 팀 비교 영역

- 아군 5명 / 적군 5명 분리
- 각 참가자 카드 표시

### 참가자 카드 핵심 정보

- Riot ID
- 챔피언 한글명
- 포지션 한글명
- K / D / A
- 승패
- 딜량
- 받은 피해량
- 골드
- 총 CS
- 시야 점수
- 와드 설치 / 제거
- 아이템 이름 목록
- 소환사 주문 이름
- 주 룬 / 보조 룬

### UX 메모

- 초기에는 팀 전체 테이블보다 `참가자 카드형 + 확장형 상세`가 읽기 쉽다.
- 현재 사용자 본인 `puuid`와 일치하는 참가자는 강조 표시한다.
- 수치가 많은 만큼 `전투`, `성장`, `시야`, `세팅` 섹션으로 나누는 것이 좋다.

### API 연결

- `GET /api/v1/matches/{matchId}`

---

## 5.4 고급 검색 화면

### 목표

- 메인 사용자 흐름을 방해하지 않고, 조건 검색을 별도 공간에서 제공한다.

### 노출 방식

- 상단 네비게이션의 서브 메뉴
- 또는 소환사 대시보드의 `고급 검색` 버튼

### 기본 필터

- 챔피언
- 포지션
- 승패
- 큐
- 최소 KDA

### 고급 필터

- 아이템
- 룬
- 스펠
- 딜량
- 골드
- 총 CS
- 시야 점수

### API 연결

- `GET /api/v1/search/matches`

### UX 메모

- 고급 검색은 초기 메인 홈보다 실험/확장 공간에 가깝다.
- 사용자 대부분은 대시보드와 경기 상세를 더 자주 사용할 가능성이 높다.

---

## 6. API와 화면 매핑

| 화면 | API | 목적 |
| --- | --- | --- |
| 홈 | `POST /api/v1/summoners/sync` | Riot ID 기준 동기화 시작 |
| 홈 | `GET /api/v1/summoners/by-riot-id` | 기존 동기화 사용자 재진입 |
| 소환사 대시보드 | `GET /api/v1/summoners/{puuid}` | 프로필 정보 표시 |
| 소환사 대시보드 | `GET /api/v1/summoners/{puuid}/matches` | 최근 경기 목록 표시 |
| 매치 상세 | `GET /api/v1/matches/{matchId}` | 경기/참가자 상세 표시 |
| 고급 검색 | `GET /api/v1/search/matches` | 조건 검색 |

프론트는 모든 API를 `api-gateway` 기준으로 호출한다.

## 7. 상태 관리 기준

### 서버 상태

- `TanStack Query` 사용
- 대상:
  - 소환사 프로필
  - 최근 경기 목록
  - 매치 상세
  - 검색 결과

### 클라이언트 상태

- 초기에는 React state로 충분
- 필요 시만 전역 상태 추가
- 초기 전역 상태 후보:
  - 최근 조회한 소환사 목록
  - 검색 필터 임시 상태

---

## 8. 비주얼 방향

### 톤

- 지나치게 게임 UI를 복제하지 않는다.
- `기록 / 회고 / 분석` 느낌이 나는 정보형 인터페이스를 지향한다.

### 권장 방향

- 폰트: `Pretendard Variable` 또는 `SUIT Variable`
- 컬러:
  - 배경: 중성 회색/미색 기반
  - 강조: 청록/딥블루 계열
  - 승/패 구분은 과한 빨강/파랑 대신 절제된 색 사용
- 레이아웃:
  - 홈은 큰 검색 입력 중심
  - 대시보드는 카드형 리스트
  - 상세는 섹션형 정보 카드

---

## 9. 구현 우선순위

### 1차 MVP

1. 홈 / 소환사 검색
2. 소환사 대시보드
3. 매치 상세

### 2차

4. 간단 필터
5. 최근 조회한 소환사 목록
6. 고급 검색 화면

### 3차

7. 패턴 분석 카드
8. AI 회고/코칭 카드

---

## 10. 한 줄 요약

> RiftMind 프론트의 초기 핵심은 `소환사 검색 -> 최근 경기 목록 -> 경기 상세 회고` 흐름이며, 검색은 보조 기능으로 두고 이후 패턴 분석과 AI 회고 카드로 확장한다.
